# doc writer

这个文档是用于记录我如何使用 vscode 和相关插件配置一个文档编辑器, 也许还有跟其他笔记软件的一些对比
还有一些提高效率的快捷键速查记录

## vs 插件

### markdownlint

来自于 vscode 对于文档写作配置区的推荐

可以用于检查我的 markdown 中有没有出现一些语法问题, 我用它来规范我的 md 文档写作习惯  
但是它的检查和我的习惯稍微有一些冲突, 比如标题前后的空行, 因为有时候我会在 vscode 中直接阅读 md 文档, 这时候大量的空行有一点不利于文字段落形成紧密的结构  
例如我习惯在 headline 前空行, 后面则直接接上这个章节的一些概述性内容, 但是 markdownlint 会要求我在前后都进行空行

我还有一些写作习惯是由于 panda 转 word 形成的, 为了形成合适的层级结构并且应用需要的格式, 我大量使用双空格的强行换行`<br>`, 这比起空行形成的行间距会窄得多, 更能够形成紧密的文字结构

安装了 markdownlint 之后, 在我的 vscode 右下角的文件语言模式提示器中显示现在 markdown 有多个不同的格式化程序, 应该设置一个为默认格式化程序, 我选择了 markdown all in one , 不知道会不会对 lint 形成什么影响, 故在此留下一个扣

### Word Count

来自 vscode 对于文档写作配置区的推荐

可以用于计算文档内有多少字, 但似乎是按照 words 计算, 不知中文字符的计算方式是怎样的

通过阅读仓库首页的说明, 发现这是微软用于演示一个 vscode 插件应该具备的设计理念的简单小功能, 那么这也意味着它没有特别的更新和适配, 大概只能用于英文吧  
我只好再去查找其他的相关插件了

### Word Count CJK

一个国人制作的字符统计插件, 现在可以计算中文字符的数量了  
命名为CJK似乎是因为中文C, 日文J, 韩文K都是在同一片unicode字符集当中, 他们之间相互区分会比较困难  
不过我已经在插件列表里面看到有能够进行区分的插件了, 另外还有中文符号转英文的插件, 对于博客的发布而言确实挺有帮助, 但使用 vscode 自带的替换功能也能够实现就是了

### Read Time

来自 vscode 对于文档写作配置区的推荐

可以用于估算阅读该文档所需要的时长, 似乎默认是根据 200 wpm 进行计算的  
我觉得使用它可以像我在众多博客中看到的那样, 可以明确文章长度和阅读时间, 想来还挺有趣  
但安装之后发现似乎没有办法有效地进行计时, 没有对中文字符进行适配  
有空可以尝试研究一下能否通过配置将字符计算来源进行修改, 但是 markdown preview enhanced 已经提供了阅读时间估计了, 姑且先不折腾了

## vs code 快捷键速查

### 跳转

| 功能                 | 键位               | 注释 |
| -------------------- | ------------------ | ---- |
| 左右编辑器组切换     | ctrl + 1, 2        |      |
| 左右编辑器组互换位置 | Ctrl + K ->        |      |
| 下一个标签页         | Ctrl + Tab         |      |
| 下一个标签页         | Ctrl + shift + Tab |      |
| 标签页跳转           | Alt + 1, 2         |      |
| 跳转到页首           | Ctrl + Home        |      |
| 标签页跳转           | Ctrl + End         |      |
| 行号跳转             | Ctrl + g           |      |
| 跳转语义块           | Ctrl + Shift + O   |      |

### 格式编辑

| 功能               | 键位                    | 注释 |
| ------------------ | ----------------------- | ---- |

| 新开空行           | Shift + Alt + UP / DOWN |      |
| 整行删除           | Ctrl + Shift + K        |      |
