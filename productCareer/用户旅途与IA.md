## 用户旅途

### 我自己的分类

我自己的分类方法, 还相当粗糙, 只是功能点 + 简单操作流

```text
- 发布帖子
  - 进入编辑页面 -> 编写带有多媒体内容的帖子 -> 发布帖子
  - 进入个人发帖记录 -> 进入已有帖子浏览界面 -> 进入编辑页面 -> 修改内容 -> 发布修改
- 浏览帖子
  - 进入内容流主页 -> 上下滑动 -> 进入帖子浏览界面 -> 浏览并发布评论 -> 回到主页
- 消息通知
  - 其他人回复帖子或评论 -> 产生通知 -> 点击后跳转至对应位置 -> 浏览并回复
- 个人主页
  - 修改个人信息
  - 修改相关设置
  - 查看个人帖子
```

### GPT 的修改

提取一级页面再修改用户旅途

1️⃣ 内容流（社区主页）
用户进入 App 后默认看到的页面
用户旅途（优化后）

进入【内容流主页】

按时间 / 热度 / 推荐排序

上下滑动浏览帖子摘要

标题

封面图 / 视频预览

作者信息

点击某一帖子

进入【帖子详情页】

浏览正文与多媒体内容

浏览评论区

点赞 / 回复评论

返回【内容流主页】

👉 这里明确区分了两个页面：

内容流主页（列表）

帖子详情页（单帖）

这是后面“页面分级”的关键。

2️⃣ 发帖与创作
内容生产相关的完整流程
用户旅途（优化后）

进入【发帖入口】（主页按钮 / 圈子页按钮）

进入【编辑页面】

输入标题

输入正文

添加多媒体内容

图片 / 视频 / 音频（可多选）

设置帖子属性

选择圈子 / 话题标签

可见范围（公开 / 圈内）

点击【发布】

发布成功，跳转至【帖子详情页】

编辑已有帖子

进入【个人主页】

进入【我的帖子】

点击某一帖子 → 进入【帖子详情页】

点击【编辑】

修改内容 / 多媒体

保存并发布修改

👉 改进点说明（你实验里可以写）：

区分「创作入口」与「内容管理入口」

引入「帖子属性」，增强社区感

3️⃣ 帖子详情与互动
围绕单一帖子的深度行为

用户旅途（补充）

在帖子详情页：

点赞帖子

收藏帖子

分享帖子

发布评论

文字评论

回复他人评论（二级评论）

删除 / 编辑自己的评论

👉 这一部分非常适合单独作为一个功能模块

4️⃣ 消息与通知
所有被动触达用户的信息

用户旅途（优化后）

系统检测到以下事件：

他人回复我的帖子

他人回复我的评论

他人点赞 / 收藏（可选）

生成消息通知

在【消息中心】显示未读提醒

用户点击通知

跳转至对应帖子或评论位置

浏览上下文内容

选择是否回复

5️⃣ 个人主页（我的）
身份、历史内容、偏好设置

用户旅途（优化后）

进入【个人主页】

查看头像、昵称、简介

修改个人信息

修改偏好设置

通知开关

隐私设置

查看内容记录
