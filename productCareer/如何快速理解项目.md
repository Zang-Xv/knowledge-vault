# 快速弄懂项目结构与逻辑（认知压缩策略）

## 目标

在 1–2 小时内，理解项目「做什么、怎么运行、关键流程在哪」，而非细节实现。

## 步骤

### 阶段 1：全局扫描

打开项目结构（VSCode / GitHub 视图）
只看这 4 类文件：

- README.md（运行命令和功能说明）
- requirements.txt 或 pyproject.toml（依赖）
- main.py / app.py（程序入口）
- routes/、models/、services/ 文件夹（按功能组织的代码）

写下你的初步判断：

功能：

- 用户登录
- 数据查询接口
- Swagger 文档页面

启动路径：
main.py → app/routers/api.py → controller.py

### 阶段 2：用模型辅助你做「结构讲解」

提示词模板：
我找到一个 GitHub 项目，文件结构如下（粘贴结构或截图文本）。
请你帮我概括它的模块职责，指出程序可能的入口点（main 文件或 app 入口），并解释代码的执行流程可能是怎样的。
输出请以「模块 → 功能 → 调用关系」的方式列出。
你可以只粘贴目录树：

```bash
app/
├── main.py
├── routers/
│   ├── users.py
│   └── items.py
├── models/
│   └── database.py
└── utils.py
```

模型会帮你「补全逻辑」并指出入口。

### 阶段 3：关键代码分析（针对主干）

挑出几个核心文件，逐段让模型帮你解释。

- 它的主要功能
- 它与项目其他模块可能的关系
- 关键逻辑流程（可以用伪代码或步骤描述）
